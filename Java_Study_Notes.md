# Chapter 01. 자바를 시작하기 전에

## 자바의 3대 핵심 특징

### 1. 객체지향 언어 (OOP, Object-Oriented Programming)

자바는 철저하게 객체 중심으로 언어가 설계되었다.
복잡한 프로그램이라도 '객체'라는 부품 단위로 쪼개어 만들고 서로 엮어서 큰 프로그램을 조립해 낸다. 이렇게 짜면 나중에 유지보수가 엄청나게 쉬워지고 코드 재사용성이 하늘을 찌른다.
(이 파트가 왜 중요한지, OOP의 실체가 뭔지는 나중에 6장~7장 객체지향 파트에서 제일 깊게 파고들 예정이다. 일단은 부품 조립식이라는 것만 알자.)

(HulaHoop 백엔드 작성 예시)
HulaHoop 백엔드 구조를 짤 때 하나의 파일에 통째로 적지 않고, Controller, Service, Mapper, DTO 등 역할별로 객체를 분담시키고 조립해서 MVC 패턴을 구상한 것이 바로 이 객체지향 방식이다.

### 2. 운영체제 독립적 (JVM 덕분)

자바 코드는 윈도우, 맥, 리눅스 어디서든 단 1줄도 수정 없이 똑같이 돌아간다. ("Write once, run anywhere")
이게 가능한 이유는 자바 프로그램이 운영체제(OS)랑 직접 대화하는 게 아니기 때문이다. 모든 운영체제 위에 자바 가상 머신(JVM, Java Virtual Machine)이라는 통역사가 깔려 있고, 내가 짠 코드는 오로지 JVM 위에서만 논다.
운영체제마다 통역을 다르게 해주는 건 JVM의 역할일 뿐, 내 코드는 바뀔 필요가 없다.

### 3. 자동 메모리 관리 (Garbage Collection, GC)

C/C++ 같은 예전 언어는 다 쓴 메모리 공간(변수, 객체)을 개발자가 일일이 코드로 삭제해 줘야 했다. 한번 놓치면 메모리가 가득 차 서버가 뻗는다.
하지만 자바는 가비지 컬렉터(GC, Garbage Collector)라는 환경미화원 시스템이 백그라운드에서 실시간 모니터링을 돌고 있다.
아무도 안 쓰는 쓰레기 데이터들을 자기가 알아서 싹 치워준다. 덕분에 개발자는 메모리 반납 같은 쓸데없는 코드에서 해방되고 비즈니스 로직(실제 기능 구현)에만 집중할 수 있다.

## 자바 실행 흐름 메커니즘

위 3가지 특징이 맞물려 현실에서 자바가 돌아가는 3단계 원리:

1. 개발자가 Java 코드 작성: 우리가 치는 소스 코드 (.java 파일 오픈)
2. 컴파일러가 번역: 자바 컴파일러(javac)가 사람 코드를 보고 컴퓨터, 즉 JVM이 읽기 좋은 '바이트코드(Bytecode)'라는 중간 언어로 싹 변환해 낸다 (.class 파일 탄생)
   - 참고: javac는 Java Compiler의 약자로, 우리가 짠 텍스트 파일(.java)을 기계(JVM)가 읽을 수 있도록 번역기 역할을 하는 프로그램 이름이다.
3. 가상머신이 실행: 이 .class 파일을 바로 그 핵심 통역사, JVM이 한 줄씩 읽어내려 가며 해당 OS의 진짜 기계어로 바꿔서 최종 실행시킨다.

결론:
운영체제가 내 코드를 직접 돌리지 않는다. 자바 코드는 튼튼한 가상 머신(JVM)이라는 쿠션 위에서 돌아가며, 필요 없어진 쓰레기는 GC가 다 치워주는 객체지향(OOP) 언어다.

# Chapter 02. 변수 

## 변수란

값을 저장해두는 공간이다.

```java
int age = 20;
```

메모리 어딘가에 공간을 잡고, 그 공간에 값을 넣어두는 개념이다.

## 자료형 (Data Type)

### 기본형 (Primitive Type)

| 타입 | 설명 |
|---|---|
| int | 정수 |
| long | 더 큰 정수 |
| double | 실수 |
| float | 실수 (double보다 작음) |
| char | 문자 1개 |
| boolean | true / false |

(Blue 백엔드 작성 예시: DTO 기본 속성 정의)
```java
int memberAge = 25;
boolean isWithdrawn = false; // 회원 탈퇴 여부
```

### 참조형 (Reference Type)

객체를 가리키는 타입이다 (배열, 클래스 등).
실제 데이터가 아니라 그 데이터의 주소(참조)를 저장한다는 점이 기본형과 다르다.

(Blue 백엔드 작성 예시)
```java
// String과 DTO 같은 것들은 전부 실제 값이 아니라 메모리 번호 주소가 남는 참조형 데이터다.
String memberId = "testUser123";
MemberDTO myInfo = new MemberDTO();
```

## 변수 선언과 초기화

선언만 따로 할 수도 있고

```java
int number;
number = 10;
```

한 번에 처리할 수도 있다.

```java
int number = 10;
```

## 상수 (final)

한 번 정하면 절대 못 바꾸는 변수다.

```java
final int MAX = 100;
```

(Blue 백엔드 작성 예시: 권한 같은 고정 상수)
```java
final String ROLE_ADMIN = "ADMIN_USER"; // 권한 관리용 상수 처리 문자열
```

# Chapter 03. 연산자

## 산술 연산자

| 연산자 | 의미 |
|---|---|
| + | 덧셈 |
| - | 뺄셈 |
| * | 곱셈 |
| / | 나눗셈 |
| % | 나머지 |

정수끼리 나눗셈 주의점:

```java
int result = 10 / 3; // 결과: 3 (소수점 버려짐)
```

## 증가 / 감소 연산자

```java
a++; // 1 증가
a--; // 1 감소
```

전위(++a)는 먼저 증가하고 쓰고, 후위(a++)는 먼저 쓰고 나중에 증가한다.

## 비교 연산자

| 연산자 | 의미 |
|---|---|
| > | 크다 |
| < | 작다 |
| >= | 이상 |
| <= | 이하 |
| == | 같다 |
| != | 다르다 |

결과는 항상 boolean 값으로 나온다.

## 논리 연산자

| 연산자 | 의미 |
|---|---|
| && | AND (둘 다 true여야 true) |
| || | OR (하나라도 true면 true) |
| ! | NOT (반전) |

(Blue 백엔드 작성 예시: 복합 조건 체크 로직)
```java
if(password != null && password.equals("1234")) {
    // 패스워드가 존재하고(AND) 동시에 1234 문자열과 똑같을 때만 블록 안으로 입장
}
```

## 대입 연산자

```java
a += 10; // a = a + 10
a -= 5;  // a = a - 5
a *= 2;  // a = a * 2
a /= 3;  // a = a / 3
```

## 형변환 (Casting)

```java
double d = 10.5;
int i = (int) d; // 10 (소수점 버려짐)
```

작은 타입 → 큰 타입은 자동 변환되지만,
큰 타입 → 작은 타입은 데이터 손실 가능성이 있어서 명시적으로 형변환해줘야 한다.

# Chapter 04. 조건문과 반복문

자바의 정석 4판 기준, 조건문과 반복문이 4장으로 통합되어 있다.

## 4-1. 조건문

### if문

```java
if (조건) {
    실행문
}
```

### if - else

```java
if (조건) {
    실행1
} else {
    실행2
}
```

### else if

여러 조건을 순서대로 검사할 때 쓴다.

```java
if (score >= 90) {
    // A
} else if (score >= 80) {
    // B
} else {
    // C
}
```

조건은 위에서부터 차례로 확인하고, 맞는 게 있으면 거기서 멈춘다.

(Blue 백엔드 작성 예시: 회원가입 중복 ID 로직)
```java
if(memberService.checkIdExists(id)) {
    // 이미 존재하는 아이디면 에러 처리
} else {
    // 사용 불가능한 조건이 아니므로 정상적으로 회원가입 프로세스 진행
}
```

### switch문

```java
switch (menu) {
    case 1:
        // 1번 선택했을 때
        break;
    case 2:
        // 2번 선택했을 때
        break;
    default:
        // 해당하는 case 없을 때
}
```

break를 빠뜨리면 그 아래 case까지 다 실행되니까 주의해야 한다.

---

## 4-2. 반복문

### for문

횟수가 정해져 있을 때 주로 사용한다.

```java
for (초기식; 조건식; 증감식) {
    실행문
}
```

가장 기본적인 10번 반복 예시:

```java
for (int i = 0; i < 10; i++) {
    // 10번 반복 내용
}
```

### while문

조건이 true인 동안 계속 반복한다.

```java
while (조건) {
    실행문
}
```

무한 루프 만들 때 쓰기도 하는데 빠져나오는 조건을 꼭 넣어야 한다.

```java
while (true) {
    if (종료조건) {
        break;
    }
}
```

### break 와 continue

- break: 반복문을 즉시 빠져나온다.
- continue: 현재 반복만 건너뛰고 다음 반복으로 넘어간다.

### 중첩 반복문

반복문 안에 반복문을 넣을 수 있다. 구구단이나 피라미드 별 출력이 대표적이다.

# Chapter 05. 배열 (Array)

## 1. 배열이란?

같은 타입의 변수 여러 개를 하나의 묶음으로 길게 나열해 놓은 데이터 저장 방식이다.
반 학생 30명의 수학 점수를 각각의 변수 30개로 각각 선언하면 코드가 폭발해버리기 때문에, 점수배열 하나로 묶어서 쉽게 관리하려고 쓴다.

## 2. 배열 선언과 생성

자바에서 배열은 그냥 숫자가 아니라 객체, 즉 참조형 타입 취급이다.

```java
// 선언과 동시에 길이(칸 갯수)를 5로 지정해서 생성
int[] score = new int[5]; // 정수 5칸짜리 기차(배열)를 메모리에 만듦
```

이렇게 하면 score[0]부터 score[4]까지 5개의 빈 방이 생기고, 자동 초기값인 0이 다 들어간다.

## 3. 배열 인덱스 (Index)

배열의 각 룸 번호를 인덱스라고 부른다.
가장 중요한 점: 1이 아니라 무조건 0부터 센다. 방이 5개면 번호는 0, 1, 2, 3, 4다. (5는 아예 없다)

```java
score[0] = 100; // 첫 번째 칸에 100 넣기
score[4] = 90;  // 마지막(5번째) 칸에 90 넣기
```

가장 많이 하는 실수로, 존재하지도 않는 score[5] 같은 걸 꺼내려 하면 ArrayIndexOutOfBoundsException 에러를 살벌하게 뱉으며 프로그램이 터진다.

## 4. 배열의 길이

배열은 한번 길이를 딱 정해서 만들고 나면(new int[5]), 프로그램 돌리는 도중에는 절대 방이나 길이를 늘리거나 줄일 수 없다. (불변성)
내가 가진 배열의 길이를 알고 싶으면 자바가 제공하는 .length 속성을 쓰면 된다.

```java
for(int i = 0; i < score.length; i++) {
    System.out.println(score[i]);
}
```
보통 for문 횟수에 하드코딩으로 숫자 5를 적지 말고 무조건 배열이름.length로 맞추는 게 정석이다. 그래야 나중에 배열 크기를 7로 고쳐도 반복문 코드를 찾아서 5를 7로 바꾸는 뻘짓을 피할 수 있다.

(Blue 백엔드 작성 예시: 프론트에서 배열로 받은 좌석 DB처리)
```java
// 프론트에서 영화 좌석 3개를 골라서 서버로 보낼 때 배열로 받는다.
String[] selectedSeats = {"A1", "A2", "A3"};

// 반복문으로 좌석들을 하나씩 DB에 인서트할 때 방어적으로 length 사용
for(int i = 0; i < selectedSeats.length; i++) {
    reservationService.insertSeat(selectedSeats[i]);
}
```

## 5. 배열 초기화

일일이 인덱스 번호를 지정해서 초기화할 수도 있지만, 처음 만들 때 중괄호 안에서 쉼표로 한 번에 처리할 수도 있다.

```java
int[] score = { 100, 90, 80, 70, 60 };
```

## 6. String 배열 (문자열 배열)

문자열도 당연히 여러 개 묶음(배열)으로 만들 수 있다. 다만 String은 기본형이 아니라 객체라서 차이가 좀 난다.

```java
String[] name = {"Kim", "Park", "Lee"};
```
기본형 숫자 배열은 아무것도 안 넣으면 0이지만, 참조형인 String 배열은 아무것도 안 넣으면 그냥 null값이 들어가는 게 큰 차이다.

## 7. 2차원 배열

배열 안의 요소가 또 배열인 형태다. 기차 안에 또 기차칸이 세워져 있어 마치 행/열로 엮인 엑셀 표 같은 모양이다.

```java
// 세로 2줄, 가로 3칸짜리 표 모양 배열 생성
int[][] score = new int[2][3];
```

알고리즘 문제 풀 때마다 이중 for문을 돌려야 해서 머리가 상당히 아파지는 단골 파트다.
