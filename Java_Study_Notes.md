# Java Study Notes (자바의 정석 1~5장 정리)

## 개요

자바 기초 문법을 처음 학습하면서 정리한 개인 스터디 기록이다.
범위는 자바의 정석 1장부터 5장까지고,
자바 기본 개념, 변수와 타입, 연산자, 조건문, 반복문, OOP 기초 순으로 진행했다.

---

# 1장. 자바 시작하기

## 프로그래밍이란

컴퓨터한테 일을 시키기 위해 명령을 작성하는 것이고,
이 명령들의 집합을 프로그램이라고 한다.

## 자바의 특징

- 객체지향 언어 (OOP)
- 운영체제 독립적 (JVM 기반으로 돌아감)
- 자동 메모리 관리 (Garbage Collection이 알아서 처리)

## 자바 실행 흐름

1. Java 코드 작성 (.java 파일)
2. 컴파일 → 바이트코드로 변환 (.class 파일)
3. JVM이 .class 파일을 읽고 실행

운영체제가 직접 실행하는 게 아니라 JVM 위에서 실행되기 때문에, 어떤 OS에서든 같은 결과가 나온다.

# 2장. 변수와 자료형

## 변수란

값을 저장해두는 공간이다.

```java
int age = 20;
```

메모리 어딘가에 공간을 잡고, 그 공간에 값을 넣어두는 개념이다.

## 자료형 (Data Type)

### 기본형 (Primitive Type)

| 타입 | 설명 |
|---|---|
| int | 정수 |
| long | 더 큰 정수 |
| double | 실수 |
| float | 실수 (double보다 작음) |
| char | 문자 1개 |
| boolean | true / false |

### 참조형 (Reference Type)

객체를 가리키는 타입이다 (배열, 클래스 등).
실제 데이터가 아니라 그 데이터의 주소(참조)를 저장한다는 점이 기본형과 다르다.

## 변수 선언과 초기화

선언만 따로 할 수도 있고

```java
int number;
number = 10;
```

한 번에 처리할 수도 있다.

```java
int number = 10;
```

## 상수 (final)

한 번 정하면 절대 못 바꾸는 변수다.

```java
final int MAX = 100;
```

# 3장. 연산자

## 산술 연산자

| 연산자 | 의미 |
|---|---|
| + | 덧셈 |
| - | 뺄셈 |
| * | 곱셈 |
| / | 나눗셈 |
| % | 나머지 |

정수끼리 나눗셈 주의점:

```java
int result = 10 / 3; // 결과: 3 (소수점 버려짐)
```

## 증가 / 감소 연산자

```java
a++; // 1 증가
a--; // 1 감소
```

전위(`++a`)는 먼저 증가하고 쓰고, 후위(`a++`)는 먼저 쓰고 나중에 증가한다.

## 비교 연산자

| 연산자 | 의미 |
|---|---|
| > | 크다 |
| < | 작다 |
| >= | 이상 |
| <= | 이하 |
| == | 같다 |
| != | 다르다 |

결과는 항상 `boolean` 값으로 나온다.

## 논리 연산자

| 연산자 | 의미 |
|---|---|
| && | AND (둘 다 true여야 true) |
| \|\| | OR (하나라도 true면 true) |
| ! | NOT (반전) |

## 대입 연산자

```java
a += 10; // a = a + 10
a -= 5;  // a = a - 5
a *= 2;  // a = a * 2
a /= 3;  // a = a / 3
```

## 형변환 (Casting)

```java
double d = 10.5;
int i = (int) d; // 10 (소수점 버려짐)
```

작은 타입 → 큰 타입은 자동 변환되지만,
큰 타입 → 작은 타입은 데이터 손실 가능성이 있어서 명시적으로 형변환해줘야 한다.

# 4장. 조건문

## if문

기본 구조

```java
if (조건) {
    실행문
}
```

## if - else

```java
if (조건) {
    실행1
} else {
    실행2
}
```

## else if

여러 조건을 순서대로 검사할 때 쓴다.

```java
if (score >= 90) {
    // A
} else if (score >= 80) {
    // B
} else {
    // C
}
```

조건은 위에서부터 차례로 확인하고, 맞는 게 있으면 거기서 멈춘다.

## switch문

```java
switch (menu) {
    case 1:
        // 1번 선택했을 때
        break;
    case 2:
        // 2번 선택했을 때
        break;
    default:
        // 해당하는 case 없을 때
}
```

`break`를 빠뜨리면 그 아래 case까지 다 실행되니까 주의해야 한다.

# 5장. 반복문

## for문

횟수가 정해져 있을 때 주로 사용한다.

```java
for (초기식; 조건식; 증감식) {
    실행문
}
```

가장 기본적인 10번 반복 예시:

```java
for (int i = 0; i < 10; i++) {
    // 반복할 내용
}
```

## while문

조건이 true인 동안 계속 반복한다.

```java
while (조건) {
    실행문
}
```

무한 루프 만들 때 쓰기도 하는데 빠져나오는 조건을 꼭 넣어야 한다.

```java
while (true) {
    if (종료조건) {
        break;
    }
}
```

## break

반복문을 즉시 빠져나온다.

```java
if (i == 5) {
    break;
}
```

## continue

현재 반복만 건너뛰고 다음 반복으로 넘어간다.

```java
if (i == 3) {
    continue;
}
```

## 중첩 반복문

반복문 안에 반복문을 넣을 수 있다.

피라미드 별 출력이 대표적인 예제인데, 핵심 공식은 아래와 같다.

- 공백 개수 = 전체 높이 - 현재 줄 번호
- 별 개수 = 현재 줄 번호 * 2 - 1

# OOP (객체지향 프로그래밍) 기초

## 객체지향이란

객체 중심으로 프로그램을 구성하는 방식이다.
현실 세계의 사물을 객체(Object)로 보고, 객체들 간의 관계로 프로그램을 설계한다.

## 핵심 개념 4가지

### 캡슐화 (Encapsulation)

데이터를 직접 건드리지 않고, 메서드를 통해서만 접근하게 제한하는 것이다.

```java
private int balance; // 외부에서 직접 접근 불가

public void deposit(int money) {
    if (money > 0) {
        balance += money;
    }
}
```

데이터 보호 + 잘못된 값이 들어오는 걸 막을 수 있음.

### 상속 (Inheritance)

기존 클래스를 그대로 가져다 쓰면서, 거기에 새로운 기능을 더하는 것이다.
중복 코드를 줄이는 데 효과적이다.

### 다형성 (Polymorphism)

하나의 타입으로 여러 형태의 객체를 다룰 수 있다는 개념이다.
같은 메서드를 호출해도 실제 객체에 따라 다른 결과가 나온다.

### 추상화 (Abstraction)

복잡한 내부 구현은 숨기고, 사용하는 쪽에서는 필요한 기능만 보이게 하는 것이다.
